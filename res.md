## ⚡ EcoFlow to Prometheus Exporter\n\nThis repo is a **Prometheus metrics exporter for EcoFlow portable power stations** (e.g., DELTA, RIVER product lines). It bridges EcoFlow's private MQTT broker and a Prometheus-compatible metrics endpoint, enabling real-time monitoring and dashboarding of EcoFlow devices.\n\n---\n\n### How It Works\n\nThe core logic lives in [`ecoflow_exporter.py`](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py) and follows a three-stage pipeline:\n\n#### 1. Authentication (`EcoflowAuthentication`)\n- [Logs into `api.ecoflow.com`](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py#L42) using the user's EcoFlow account credentials (username/password).\n- [Fetches MQTT credentials](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py#L62) (certificate account + password) from the IoT auth endpoint — the same flow used by EcoFlow's official mobile app.\n- No need for a separately requested `APP_KEY`/`SECRET_KEY`.\n\n#### 2. MQTT Subscription (`EcoflowMQTT`)\n- [Connects to `mqtt.ecoflow.com:8883`](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py#L34) over TLS.\n- [Subscribes to the device topic](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py#L108): `/app/device/property/{device_sn}`.\n- Incoming MQTT payloads are pushed into a thread-safe `Queue`.\n- Includes an [idle reconnect watchdog](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py#L136): if no messages arrive within a configurable timeout, it spawns a subprocess to reconnect (with a 60-second hard timeout per attempt).\n\n#### 3. Metric Processing & Export (`Worker` + `EcoflowMetric`)\n- The `Worker` [polls the queue every N seconds](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py#L234). If no messages arrive, the device is marked **offline** and all metrics are **cleared** (so Prometheus sees NaN rather than stale values).\n- For each key/value pair in the JSON payload, an [`EcoflowMetric`](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py#L191) Prometheus `Gauge` is **dynamically created on first sight** — no hardcoded metric list required.\n- EcoFlow camelCase/dot-notation keys are [converted to Prometheus snake_case names](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py#L198): e.g., `bms_bmsStatus.maxCellTemp` → `ecoflow_bms_bms_status_max_cell_temp`.\n- All metrics carry a `device` label (serial number or a custom name), enabling **multi-device support**.\n- One known [data-correction hack](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/ecoflow_exporter.py#L295): AC input current is zeroed when AC input voltage is zero (device firmware quirk).\n- A Prometheus HTTP server is started via `prometheus_client.start_http_server`.\n\n---\n\n### Key Design Choices\n\n| Aspect | Detail |\n|---|---|\n| **Protocol** | MQTT over TLS (port 8883), mimicking the official EcoFlow app |\n| **Metric creation** | Fully dynamic — new payload keys auto-generate new Prometheus gauges |\n| **Offline detection** | Empty queue → `ecoflow_online=0` + metrics cleared |\n| **Resilience** | Idle reconnect timer runs in a subprocess with a 60s timeout guard |\n| **Configuration** | Entirely via environment variables (`DEVICE_SN`, `ECOFLOW_USERNAME`, `ECOFLOW_PASSWORD`, etc.) |\n\n---\n\n### Ecosystem\n\n- **[`Dockerfile`](https://github.com/berezhinskiy/ecoflow_exporter/blob/b0fdd89a5cf6/Dockerfile)** — pre-built Docker image published to GitHub Container Registry.\n- **[`docker-compose/`](https://github.com/berezhinskiy/ecoflow_exporter/tree/b0fdd89a5cf6/docker-compose)** — a full-stack quick-start with Prometheus, Grafana, and Alertmanager (including Telegram alert templates).\n- **[Grafana Dashboard](https://grafana.com/grafana/dashboards/17812-ecoflow/)** — a published dashboard for visualizing all exported metrics.\n- Tested with: **RIVER Max, RIVER 2, RIVER 2 Max, DELTA Max, DELTA 2, DELTA 2 Max, DELTA Pro**.
